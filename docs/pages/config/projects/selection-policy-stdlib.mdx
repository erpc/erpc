# Selection policy stdlib reference

Every method on `upstreams` returns an `Upstream[]` so chains compose.
Start with the [selection policy overview](/config/projects/selection-policies)
for the model; this page is the catalogue.

Cross-tick state (`previousOrder`, `excludedSince`, `lastSwitchAt`,
`tickCount`) is read from the `ctx` argument of the outer eval. Glob
patterns (`*`, `?`, `!negation`) work everywhere a `string | string[]`
is accepted.

## Constants

**Score presets** (weight maps for `sortByScore`):

| Constant | `errorRate, respLatency, throttledRate, blockHeadLag, finalizationLag, misbehaviors` |
|---|---|
| `BALANCED` | `8, 4, 3, 2, 1, 6` |
| `PREFER_FASTER` | `4, 12, 4, 1, 0, 2` |
| `PREFER_FEWER_ERRORS` | `15, 2, 6, 2, 1, 12` |
| `PREFER_FRESHER_HEAD` | `4, 2, 2, 15, 8, 3` |
| `PREFER_LESS_THROTTLED` | `8, 4, 15, 2, 1, 4` |
| `PREFER_CHEAP` | alias of `BALANCED` |

**Finality:** `REALTIME` · `UNFINALIZED` · `FINALIZED` · `UNKNOWN` —
match values of `ctx.finality`.

**Reasons:** `REASON_LAG` · `REASON_ERROR_RATE` · `REASON_LATENCY` ·
`REASON_THROTTLING` · `REASON_MISBEHAVIOR` · `REASON_CORDONED` ·
`REASON_GROUP` · `REASON_VENDOR` · `REASON_USER_FILTER`.

## Identity & label selection

Upstreams carry an open-ended **`tags`** slice (recommended convention
`<dimension>:<value>`, e.g. `tier:main`, `region:us-east`, `sequencer:op-base`).
Stdlib tag methods read this directly:

```ts
.where(filter: { id?, tag?, vendor?, type? })      // AND across fields
.whereNot(filter)                                   // inverse
.byId(id: string | string[])
.byTag(pat: string | string[])                      // 'tier:main' / 'region:us-*' / '!tier:fallback'
.byVendor(name: string | string[])
.byType(type: string | string[])
.excludeId / .excludeTag / .excludeVendor           // negation forms
```

**Negation semantics.** `byTag('!X')` matches upstreams that have NO tag
matching `X` (it's the dual of `byTag('X')`). `excludeTag('X')` is the
same thing. Array form mixes: positive patterns are OR'd (any match wins);
negated patterns are AND'd (all must hold); both kinds together mean
`(any positive matches OR no positives in the array) AND (all negations hold)`.

**Deprecated aliases** — `byGroup(name)` / `excludeGroup(name)` /
`preferGroup(name, opts)` still work as shims for `byTag('tier:'+name)`
etc., so policies written before the tags merge keep running. New
policies should use the tag methods directly.

## Health filters

Each drops upstreams failing the predicate and annotates them for the
decision record.

```ts
.removeByErrorRate(max: number)              // drop if errorRate > max
.removeByLatency({ p50Ms?, p70Ms?, p90Ms?, p95Ms?, p99Ms? })
.removeByThrottling(max: number)
.removeByMisbehavior(max: number)
.removeByLag({ blockHead?: number, finalization?: number })
.removeByMinRequests(min: number)            // require ≥ min samples
.removeCordoned()                            // drop if cordonedReason != null
```

Composite shortcut:

```ts
.keepHealthy({
  maxErrorRate?: number     = 0.5,
  maxBlockHeadLag?: number  = 10,
  maxP95Ms?: number         = 5000,
  maxThrottledRate?: number = 0.3,
})
```

## Generic functional

Standard array combinators that operate on upstream identity (set ops
dedupe by `id`):

```ts
.filter(fn) · .reject(fn) · .partition(fn): [yes, no]
.unique(keyFn?) · .union(other) · .intersect(other) · .difference(other)
.slice(start, end?) · .reverse() · .isEmpty
```

## Sorting

Primary entry point — `sortByScore`:

```ts
.sortByScore(
  weights: ScoreWeights | preset | ((u) => ScoreWeights),
  opts?: {
    latencyQuantile?: 'p50' | 'p70' | 'p90' | 'p95' | 'p99'  // default 'p70'
    overall?: (u) => number                                   // post-multiplier
  }
)
```

`penalty = Σ(metric × weight) × overall(u)`. Lower penalty = higher
rank. Score + breakdown are attached to each upstream for later steps.

The first arg can be:
- a **preset** (`BALANCED`, `PREFER_FASTER`, ...);
- a flat **weight map** (`{ errorRate: 10, respLatency: 3 }`);
- a **per-upstream function** (`(u) => weights` — replaces legacy `scoreMultipliers`).

Other sorts (ascending unless noted):

```ts
.sortBy(fn, { desc? }) · .sortByDesc(fn)
.sortByLatency(quantile?) · .sortByErrorRate() · .sortByThrottling()
.sortByMisbehavior() · .sortByHeadLag() · .sortByFinalizationLag()
```

## Randomization & rotation

```ts
.shuffle(seed?)
.rotateBy(n)                  // left-rotate by n; pair with ctx.tickCount for RR
```

## Stability (cross-tick)

```ts
.stickyPrimary({
  hysteresis?: number = 0.10,             // challenger must be this fraction better
  minSwitchInterval?: Duration = '30s'    // cooldown between switches
})
```

Reads `ctx.previousOrder[0]` and `ctx.lastSwitchAt`. Switches only when
the challenger's `score` is `< prevScore × (1 - hysteresis)` AND the
cooldown has elapsed.

## Tiering & multi-tier

```ts
.preferTag(pat: string, opts?: { minHealthy?: number = 1, fallback?: string })
.preferVendor(name: string, opts?: same)

.spreadAcrossTags(prefix: string)
```

`preferTag` returns upstreams whose tags match `pat` (glob; `!negation`
supported) if at least `minHealthy` match; else falls through to the
`fallback` pattern; else returns the input unchanged. The default policy
uses `preferTag('!tier:fallback', { fallback: 'tier:fallback' })` to
implement the standard "primary tier first, fallback tier if empty"
behavior.

`preferVendor` is the analogous primitive against the derived `u.vendor`
attribute (not user-set; comes from the upstream type/endpoint).

`spreadAcrossTags(prefix)` *re-interleaves* an already-sorted list so
adjacent positions don't share the same tag matching `prefix`. Use it
AFTER `sortByScore` to get blast-radius diversity in the failover order
without losing the score-based selection of the primary.

```js
// Top 3 by score might all be in cohort:op-base-sequencer. Without
// spread, a sequencer outage kills primary AND the first two fallbacks
// simultaneously. With spread, position 0 = best in cohort A, position
// 1 = best in cohort B, position 2 = second-best in cohort A, etc.
upstreams.sortByScore(BALANCED).spreadAcrossTags('cohort:')
```

Upstreams without any tag matching the prefix all land in the
empty-key bucket and interleave together at the end of the partition
order.

**Deprecated.** `preferGroup(name, opts)` and `spreadAcrossGroups({ by })`
remain as shims:

- `preferGroup('main', ...)` ≡ `preferTag('tier:main', ...)`
- `spreadAcrossGroups({ by: 'cohort' })` ≡ `spreadAcrossTags('cohort:')`
- `spreadAcrossGroups({ by: 'vendor' })` ≡ partition by `u.vendor`
  (no direct tag equivalent; `vendor` isn't a tag — it's a derived
  attribute). If you need this, use `.sortByScore(...).spreadAcrossGroups({by:'vendor'})`
  for now; a future `spreadAcrossVendors()` may replace it.


## Probing & forced inclusion

```ts
.probeExcluded({
  reAdmitAfter: Duration,
  maxConcurrent?: number = 1,
  longestFirst?: boolean = true,
  position?: 'tail' | 'head' | 'random' = 'tail',
})
```

Deterministic re-admission. Each tick, picks up to `maxConcurrent`
upstreams whose `excludedSince` is older than `reAdmitAfter` (oldest
first when `longestFirst`) and inserts them at `position`.

```ts
.forceInclude(idOrFn: string | string[] | ((u) => boolean), position?: 'head' | 'tail' = 'tail')
```

Always include matching upstreams, even if prior filters dropped them.

## Cooldown

```ts
.cooldown(duration: Duration)
```

Drops upstreams whose `excludedSince[id]` is within `duration` of
`ctx.now`. Pairs naturally with `probeExcluded`.

## Combinators

```ts
.if(cond, thenFn, elseFn?)             // cond is boolean | (arr) => boolean
.unless(cond, fn)
.whenEmpty(() => Upstream[])           // run only if currently empty
.whenNotEmpty(fn)
.fallbackTo(arrOrFn)                   // replace with alternative if empty
.ensureMin(n, fn)                      // run fn to expand if length < n

.byFinality({                          // route by ctx.finality
  realtime?:    (u) => Upstream[],
  unfinalized?: (u) => Upstream[],
  finalized?:   (u) => Upstream[],
  unknown?:     (u) => Upstream[],
})
```

`byFinality` dispatches to the handler matching `ctx.finality`. A
missing handler passes through unchanged — `byFinality({ finalized: f })`
only branches on FINALIZED requests, no-op for the rest. Replaces the
recurring `if (ctx.finality === ...)` boilerplate.

```js
// Trust any upstream for finalized reads; strict for live-latest;
// fall back to balanced for everything else.
upstreams.byFinality({
  finalized: u => u.sortByScore(PREFER_CHEAP),
  realtime:  u => u.removeByLag({ blockHead: 5 }).sortByScore(PREFER_FRESHER_HEAD),
})
.sortByScore(BALANCED)                 // applies to unfinalized + unknown only
```

## Debug & annotations

```ts
.tap(fn)                               // side effect; returns arr unchanged
.label(name)                           // name this step in the decision record
.annotate(fn)                          // attach a note to each upstream
.mark(predicate, note)                 // annotate only matching upstreams
.dump(level?: 'debug' | 'info' | 'warn' | 'error')
```

```ts
methodMatches(pattern: string | string[]): boolean    // glob ctx.method
```

## Examples

### Cost-tier with weekday schedule
```js
// Upstreams declare their tier via `tags: [tier:cheap]` or `tags: [tier:fast]`.
(upstreams, ctx) => {
  const cheapHours = inWindow('09:00', '18:00') && (new Date().getDay() % 6) !== 0
  return upstreams
    .removeByErrorRate(0.5)
    .preferTag(cheapHours ? 'tier:cheap' : 'tier:fast',
               { minHealthy: 1, fallback: 'tier:cheap' })
    .sortByScore(BALANCED)
    .stickyPrimary({ hysteresis: 0.15, minSwitchInterval: '2m' })
}
```

### Per-method override
```js
// Upstreams declare their capability via `tags: [tier:archive]`.
(upstreams, ctx) => {
  if (methodMatches(['eth_getLogs', 'eth_getBlockByNumber']))
    return upstreams.byTag('tier:archive').sortByScore(PREFER_FASTER)
  if (methodMatches('eth_getBalance'))
    return upstreams.sortByScore(PREFER_FEWER_ERRORS)
  return upstreams.sortByScore(BALANCED).stickyPrimary()
}
```

### Canary with cooldown
```js
(upstreams, ctx) =>
  upstreams
    .removeCordoned()
    .removeByErrorRate(0.5)
    .cooldown('1m')
    .sortByScore(BALANCED)
    .probeExcluded({ reAdmitAfter: '5m', maxConcurrent: 1 })
    .forceInclude('canary-rpc', 'tail')
```

### Debug a flaky decision
```js
(upstreams, ctx) =>
  upstreams
    .removeByLag({ blockHead: 5 }).label('lag-filter')
    .sortByScore(BALANCED).label('score')
    .stickyPrimary().label('sticky')
    .dump('debug')
```

`label(...)` makes each step's rejections show `step: "lag-filter"` in
the decision record.

## Planned primitives (not yet shipped)

Three additional primitives are tracked in the spec but not yet in the
stdlib. Listed here so policy authors know what's coming and can defer
designing workarounds:

| Planned | Solves | Status |
|---|---|---|
| `.probeState({ method, target, slot, every, expectChange, excludeOn })` | Lying upstreams that claim a fresh block but serve `0x0` or stale state. Currently undetectable from request-side metrics; operators have to manually disable. | Background prober + DSL surface. Larger change; tracked separately. |

These are not workarounds you can build yourself today using existing
primitives — they need new behavior (cohort labels, background probes)
or new ergonomics.
