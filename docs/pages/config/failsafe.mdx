# Failsafe

A broken upstream becomes invisible to your callers. eRPC wraps every request in six
independently tunable policies — retry, hedge, timeout, circuit breaker, consensus, and
integrity — that run in a fixed, deterministic chain. Configure once per method or finality
tier; let eRPC handle the rest.

### How it works

**Executor composition.** The non-consensus executor chain is `retry(hedge(runUpstreamSweep))`; with consensus active it becomes `consensus(retry(hedge(tryOneUpstream)))`. The network-scope timeout wraps the entire `networkExecutor.Run` call — so it bounds ALL retries and hedges, not individual attempts. (<SourceLink file="erpc/network_executor.go" lines="183-203" />)

**Scope restrictions.**
- Circuit breakers are **upstream-scope only**. Placing a `circuitBreaker` block in a network-level `failsafe` entry causes startup failure with `ErrFailsafeConfiguration`. (<SourceLink file="erpc/network_executor.go" lines="68-73" />)
- Consensus is **network-scope only**. Placing a `consensus` block in an upstream-level `failsafe` entry causes startup failure. (<SourceLink file="upstream/upstream_executor.go" lines="46-53" />)

**Failsafe entry selection.** There are two ways to scope a `failsafe[]` entry to a class of requests: the unified `matchers` object (recommended) and the legacy `matchMethod` + `matchFinality` fields. They can be mixed across entries; a single entry uses whichever it sets.

**Unified `matchers` (recommended).** Set `matchers` on a `failsafe[]` entry to scope it by `network`, `method`, request `params`, and/or `finality`, with an explicit `action` of `include` (default) or `exclude`:

```yaml filename="erpc.yaml"
failsafe:
  - matchers:
      - method: "eth_getLogs"
        finality: [unfinalized, realtime]
        action: include
    hedge:
      maxCount: 2
      delay: 100ms
```

Semantics:
- Within one entry's `matchers` list, rules are evaluated **last-to-first — the last matching rule wins** — and the entry is selected when that rule's `action` is `include`. This lets you write `[{method: "*", action: exclude}, {method: "eth_getLogs", action: include}]` for a default-exclude base with a narrow include.
- `network` / `method` / string `params` use the standard wildcard grammar (globs, `|`/`&`/`!`, parentheses, `<empty>`, numeric `>=`/`<=`/`>`/`<`/`=` on hex values). `params` is positional and recursive (objects match a subset of keys, arrays match element-wise by exact length). `finality` is an OR-set of `finalized` / `unfinalized` / `realtime` / `unknown`. An unset field imposes no constraint.
- Across `failsafe[]` entries, **the first entry (in config order) whose `matchers` resolve to `include` wins.** Matcher-based entries are considered before legacy `matchMethod`/`matchFinality` entries, so in a mixed config a matching matcher entry takes precedence over a legacy entry regardless of position. (<SourceLink file="common/matcher_config.go" lines="178-216" />, <SourceLink file="erpc/networks.go" lines="835-873" />, <SourceLink file="upstream/upstream.go" lines="372-433" />)

Invalid wildcard patterns in `matchers` (e.g. an unbalanced paren) are rejected at config load rather than silently never matching.

**Legacy `matchMethod` + `matchFinality` (deprecated).** Entries without `matchers` are selected by the prior algorithm, preserved for backward compatibility:
- **Network scope:** first matching entry in config order wins (`matchMethod` wildcard + `matchFinality` set membership).
- **Upstream scope:** 4-tier priority — (1) specific method + specific finality, (2) specific method any finality, (3) wildcard method specific finality, (4) wildcard method any finality (catch-all); within a tier, first in config order wins.

**`matchFinality` valid values.** `finalized`, `unfinalized`, `realtime`, `unknown`. A value like `"latest"` is not a valid finality token and silently never matches any request. (The same finality tokens apply to a matcher's `finality` field.)

**Defaults merge algorithm.** When a network has its own `failsafe` array AND `networkDefaults.failsafe` is also set, each entry in the network array is matched against the defaults array using the `WildcardMatch + MatchFinalities` algorithm on `matchMethod`/`matchFinality`. When a match is found, unset sub-fields (retry, hedge, timeout, etc.) inherit from the matched default entry. A catch-all default (`matchMethod: "*"`) acts as a universal base for all network-specific entries. If the network has no `failsafe` entries but the defaults do, the entire defaults array is cloned. (<SourceLink file="common/defaults.go" lines="1793-1832" />)

> Note: the defaults-merge pairing keys off `matchMethod`/`matchFinality` only — it does **not** read `matchers`. A matcher-based entry keeps its `matchers` through the merge (they are never dropped) and inherits unset sub-policies from the catch-all default, but its `matchers` constraints are not used to pick which default it pairs with. If you rely on `networkDefaults.failsafe` inheritance, set the sub-policies explicitly on matcher-based entries rather than depending on per-method pairing.

**Network-scope timeout is lifecycle-scoped, not per-attempt.** A 500ms network timeout with `maxAttempts: 3` has a 500ms total budget shared across all 3 retry rounds. There is no per-attempt timeout at the network scope.

**`retryEmpty` directive is the master gate** for all empty/missing-data retries. When off (`false`, the default), neither the network scope nor the upstream scope retries empty/missing-data results, and the empty-as-error conversion hook is also suppressed. (<SourceLink file="erpc/network_executor.go" lines="401-404" />)

### Worked examples

**1. Standard production failsafe — latency-sensitive workload.** Adaptive p70 hedge fires after the method's p70 latency; network retries up to 5 times on errors; network timeout bounds the entire request lifecycle:

<ConfigTabs
  path="projects[].networks[].failsafe[]"
  yaml={`failsafe:
  - matchMethod: "*"
    timeout:
      duration: 30s
    retry:
      maxAttempts: 5
      delay: 200ms
      backoffFactor: 1.5
      backoffMaxDelay: 5s
    hedge:
      delay:
        quantile: 0.7
        min: 100ms
        max: 2s
      maxCount: 1`}
  ts={`failsafe: [{
  matchMethod: "*",
  timeout: { duration: "30s" },
  retry: {
    maxAttempts: 5,
    delay: "200ms",
    backoffFactor: 1.5,
    backoffMaxDelay: "5s",
  },
  hedge: {
    delay: { quantile: 0.7, min: "100ms", max: "2s" },
    maxCount: 1,
  },
}]`}
/>

**2. Per-method finality tuning.** `eth_getLogs` on finalized data gets a longer timeout and more retries; all other finalized methods get a tighter budget:

<ConfigTabs
  path="projects[].networks[].failsafe[]"
  yaml={`failsafe:
  - matchMethod: "eth_getLogs"
    matchFinality: [finalized]
    timeout:
      duration: 60s
    retry:
      maxAttempts: 5
  - matchMethod: "*"
    matchFinality: [finalized]
    timeout:
      duration: 10s
    retry:
      maxAttempts: 3`}
  ts={`failsafe: [
  {
    matchMethod: "eth_getLogs",
    matchFinality: ["finalized"],
    timeout: { duration: "60s" },
    retry: { maxAttempts: 5 },
  },
  {
    matchMethod: "*",
    matchFinality: ["finalized"],
    timeout: { duration: "10s" },
    retry: { maxAttempts: 3 },
  },
]`}
/>

**3. Upstream-scope circuit breaker.** Upstream-level failsafe adds an auto-healing circuit breaker; the network-level failsafe above still applies for retries and hedging:

<ConfigTabs

      failureThresholdCapacity: 80
      halfOpenAfter: 5m
      successThresholdCount: 8
      successThresholdCapacity: 200`}

    failureThresholdCount: 20,

### Best practices

- Place **circuit breakers at upstream scope only**; placing them at network scope causes `ErrFailsafeConfiguration` at startup.
- Place **consensus at network scope only**; placing it at upstream scope also causes `ErrFailsafeConfiguration` at startup.
- Always use a **catch-all defaults entry** (`matchMethod: "*"`) in `networkDefaults.failsafe` so method-specific entries inherit the base policy without repeating every field.
- The **network-scope timeout** wraps all retries and hedges — always set it shorter than `server.maxTimeout` (default 150s), or it will never fire on its own.
- Stacking `upstreams[].failsafe[].retry.maxAttempts: 3` with `networks[].failsafe[].retry.maxAttempts: 3` can produce up to 9 upstream calls per request — size budgets accordingly.
- `emptyResultMaxAttempts` (default 2) is a **shared counter across ALL network retry rounds**, not per-round. With default `maxAttempts=5`, empty-type retries are capped at 1 extra call total.
- Enable `retryEmpty: true` in `directiveDefaults` (or per-request via `X-ERPC-Retry-Empty: true`) if you need missing-data retries — it is off by default.

### Edge cases & gotchas

1. `matchFinality: ["latest"]` silently never matches. Valid values: `finalized`, `unfinalized`, `realtime`, `unknown`.
2. Stacking upstream retry (`maxAttempts: 3`) with network retry (`maxAttempts: 3`) can produce up to 9 upstream calls per request.
3. Write methods (`eth_sendTransaction`, filter-creation methods) are excluded from hedging. `eth_sendRawTransaction` is intentionally NOT excluded — it supports idempotent broadcast.
4. `emptyResultMaxAttempts` is a shared counter across all network retry rounds, not per-round. With the default value of 2, one original attempt plus one empty-type retry fires — at most 1 empty-type retry across the entire request lifetime.
5. Consensus `ignoreFields` is a set-replacement, not a merge. Setting any entry replaces the entire built-in default map (which suppresses `blockTimestamp` disagreements for `eth_getLogs` and receipts methods).
6. The circuit breaker's rolling window must fill to `failureThresholdCapacity` (default 80) before it can trip — early-startup behavior is Closed regardless of error rate.
7. `ErrFailsafeConfiguration` is startup-only — any scope mismatch (circuit breaker at network, consensus at upstream) aborts before any request is served.
8. Failsafe defaults merge uses **wildcard method matching**, not exact match. A default entry with `matchMethod: "eth_*"` will provide defaults for any network entry whose method starts with `eth_`. A catch-all `matchMethod: "*"` is required to cover all entries uniformly.
