# Selection policy stdlib reference

Every method on `upstreams` (and on the result of any prior step) returns an
`Upstream[]`, so chains compose. This page is the exhaustive catalogue —
the [selection-policy overview](/config/projects/selection-policies) is
where to start if you have not seen the new shape yet.

All methods take their `this` as the upstream array; cross-tick state (the
`previousOrder`, `excludedSince`, `lastSwitchAt`, `tickCount` fields) is
read from the `ctx` argument of the outer eval function.

## Constants

Score presets — weight maps for `sortByScore`:

| Constant | `errorRate, respLatency, throttledRate, blockHeadLag, finalizationLag, misbehaviors` |
|---|---|
| `BALANCED` | `8, 4, 3, 2, 1, 6` |
| `PREFER_FASTER` | `4, 12, 4, 1, 0, 2` |
| `PREFER_FEWER_ERRORS` | `15, 2, 6, 2, 1, 12` |
| `PREFER_FRESHER_HEAD` | `4, 2, 2, 15, 8, 3` |
| `PREFER_LESS_THROTTLED` | `8, 4, 15, 2, 1, 4` |
| `PREFER_CHEAP` | (alias of `BALANCED`; intended pairing with `preferGroup('cheap')`) |

Finality states (match values of `ctx.finality`):
`REALTIME`, `UNFINALIZED`, `FINALIZED`, `UNKNOWN`.

Reason codes used for annotation:
`REASON_LAG`, `REASON_ERROR_RATE`, `REASON_LATENCY`, `REASON_THROTTLING`,
`REASON_MISBEHAVIOR`, `REASON_CORDONED`, `REASON_GROUP`, `REASON_VENDOR`,
`REASON_USER_FILTER`.

## Identity & label selection

```ts
.where(filter: { id?, group?, vendor?, type? })
.whereNot(filter: same as above)
.byId(id: string | string[])
.excludeId(id: string | string[])
.byGroup(name: string | string[])
.excludeGroup(name: string | string[])
.byVendor(name: string | string[])
.excludeVendor(name: string | string[])
.byType(type: string | string[])
```

Filter values support glob patterns: `byGroup('primary*')`,
`byVendor('!alchemy')` (`!` = negation). Multiple fields in `.where({...})`
are AND-ed; arrays are OR-ed.

```js
upstreams.byGroup('primary*').excludeVendor('alchemy')
```

## Health filters

Each drops upstreams failing the predicate. Rejected upstreams are
annotated `{rejectedBy: <step>, reason: <human string>}` and appear in the
`excluded` list of the decision record.

```ts
.removeByErrorRate(maxRate: number)
.removeByLatency({ p50Ms?, p70Ms?, p90Ms?, p95Ms?, p99Ms? })
.removeByThrottling(maxRate: number)
.removeByMisbehavior(maxRate: number)
.removeByLag({ blockHead?: number, finalization?: number })
.removeByMinRequests(min: number)    // drop if requestsTotal < min
.removeCordoned()                    // drop if metrics.cordonedReason != null
```

Convenience composite:

```ts
.keepHealthy({
  maxErrorRate?: number     = 0.5,
  maxBlockHeadLag?: number  = 10,
  maxP95Ms?: number         = 5000,
  maxThrottledRate?: number = 0.3,
})
```

```js
upstreams.removeByLag({ blockHead: 5, finalization: 50 })
         .removeByErrorRate(0.3)
```


## Sorting

The primary scoring entry point:

```ts
.sortByScore(
  weights: ScoreWeights | preset | ((u: Upstream) => ScoreWeights),
  opts?: {
    latencyQuantile?: 'p50' | 'p70' | 'p90' | 'p95' | 'p99'  // default 'p70'
    overall?: (u: Upstream) => number   // per-upstream final-penalty multiplier
  }
): Upstream[]
```

`ScoreWeights = { errorRate?, respLatency?, throttledRate?, blockHeadLag?, finalizationLag?, misbehaviors? }`.

The first argument can be:

- a **flat weight map** — applied to every upstream;
- a **preset constant** — `BALANCED`, `PREFER_FASTER`, etc.;
- a **function `(u) => ScoreWeights`** — per-upstream weights. Useful for
  vendor-specific tuning or migrating per-upstream multipliers from the
  legacy config.

`penalty = Σ(metric × weight) × overall(u)`. Lower penalty = higher rank.
Each upstream's resulting `score` and `penaltyBreakdown` are attached and
visible to subsequent steps + decision records.

```js
// Preset
upstreams.sortByScore(BALANCED)

// Flat weights
upstreams.sortByScore({ errorRate: 10, respLatency: 3 })

// Per-upstream weights (replaces legacy scoreMultipliers)
const weightsByID = {
  alchemy:  { errorRate: 8, respLatency: 12 },
  drpc:     { errorRate: 8, respLatency: 4 },
};
upstreams.sortByScore((u) => weightsByID[u.id] || BALANCED)
```

Other sorts (ascending unless noted):

```ts
.sortBy(fn: (u) => number, opts?: { desc?: boolean })
.sortByDesc(fn: (u) => number)
.sortByLatency(quantile?: 'p50' | 'p70' | 'p90' | 'p95' | 'p99')   // default 'p70'
.sortByErrorRate()
.sortByThrottling()
.sortByMisbehavior()
.sortByHeadLag()
.sortByFinalizationLag()
```

```js
// Round-robin replacement for legacy routingStrategy: round-robin

```

## Stability (cross-tick)

```ts
.stickyPrimary({
  hysteresis?: number = 0.10,
  minSwitchInterval?: Duration = '30s'
})
```

Reads `ctx.previousOrder[0]` and `ctx.lastSwitchAt`. Switches primary only
when the challenger's `score` is at least `hysteresis` fraction lower AND
`minSwitchInterval` has elapsed since the last switch. Without a prior
`sortByScore`, falls back to positional stickiness (the prior primary is
moved to position 0 if still present).

## Grouping (multi-tier)

```ts
.preferGroup(name: string, opts?: {
  minHealthy?: number = 1,
  fallback?: string                  // if group < minHealthy, switch to this group
})
.preferVendor(name: string, opts?: same)
```

```js
upstreams.preferGroup('default', { minHealthy: 1, fallback: 'fallback' })
```

## Slicing & limits

```ts
.pickTop(n: number)        // first n
.pickBottom(n: number)
.dropTop(n: number)        // skip first n
.dropBottom(n: number)
.take(n)                   // alias for pickTop
.skip(n)                   // alias for dropTop
```

## Probing & forced inclusion

```ts
.probeExcluded({
  reAdmitAfter: Duration,
  maxConcurrent?: number = 1,
  longestFirst?: boolean = true,
  position?: 'tail' | 'head' | 'random' = 'tail'
})
```

Deterministic re-admission of excluded upstreams. Each tick, finds
upstreams in `ctx.excludedSince` older than `reAdmitAfter`, sorts by
exclusion age (oldest first when `longestFirst`), picks up to
`maxConcurrent`, and inserts them at the specified position so their
metrics refresh organically.

```ts
.forceInclude(idOrFn: string | string[] | ((u) => boolean), position?: 'head' | 'tail' = 'tail')
```

Always include matching upstream(s), bypassing prior filters. Useful for
canary/maintenance modes.

## Cooldown

```ts
.cooldown(duration: Duration)
```

Removes upstreams whose `ctx.excludedSince[id]` is within `duration` of
`ctx.now`. Pairs naturally with `probeExcluded`: keep recently-excluded
upstreams out for at least the cooldown window, then probe them back in.

## Combinators (chain control flow)

```ts
.if(cond: boolean | ((arr) => boolean), thenFn: (arr) => Upstream[], elseFn?: (arr) => Upstream[])
.unless(cond: boolean | ((arr) => boolean), fn: (arr) => Upstream[])
.whenEmpty(fn: () => Upstream[])         // run only if chain currently empty
.whenNotEmpty(fn: (arr) => Upstream[])
.fallbackTo(arrOrFn: Upstream[] | ((ctx) => Upstream[]))   // if empty, replace
.ensureMin(n: number, fn: (arr) => Upstream[])             // if length < n, run fn
```

```js
upstreams
  .keepHealthy()
  .whenEmpty(() => upstreams.byGroup('fallback'))   // emergency fallback
  .sortByScore(BALANCED)
```

## Debug & annotations

```ts
.tap(fn: (arr) => void)                    // side effect; returns arr unchanged
.label(name: string)                       // names this chain step in decision records
.annotate(fn: (u) => string)               // attach a note to each upstream
.mark(predicate: (u) => boolean, note: string)
.dump(level?: 'debug' | 'info' | 'warn' | 'error')
```

`.label` is purely for the decision record — operators see which step a
rejection came from. `.dump` writes the current chain state to the
structured logger at the given level.

## Free helpers (module-level globals)

```ts
methodMatches(patternOrPatterns: string | string[]): boolean
// Test ctx.method against glob patterns.

isFinalityRequest(): boolean
// Sugar for ctx.finality === FINALIZED.

inWindow(start: string, end: string, tz?: string): boolean
// Time-of-day window check using ctx.now (HH:MM, 24h).

durationMs(d: Duration | string): number
// Parse a duration string into ms. Used internally by stickyPrimary, etc.
```

## Examples

### Cost-tier with weekly schedule

```js
(upstreams, ctx) => {
  // Weekdays 9–18 local: drain expensive vendors.
  const cheapHours = inWindow('09:00', '18:00') && (new Date().getDay() % 6) !== 0;
  return upstreams
    .removeByErrorRate(0.5)
    .preferGroup(cheapHours ? 'cheap' : 'fast', { minHealthy: 1, fallback: 'cheap' })
    .sortByScore(BALANCED)
    .stickyPrimary({ hysteresis: 0.15, minSwitchInterval: '2m' });
}
```

### Per-method weight overrides

```js
(upstreams, ctx) => {
  if (methodMatches(['eth_getLogs', 'eth_getBlockByNumber'])) {
    return upstreams.byGroup('archive').sortByScore(PREFER_FASTER);
  }
  if (methodMatches('eth_getBalance')) {
    return upstreams.sortByScore(PREFER_FEWER_ERRORS);
  }
  return upstreams.sortByScore(BALANCED).stickyPrimary();
}
```

### Per-upstream weights (legacy `scoreMultipliers` replacement)

```js
const weights = {
  alchemy:  { errorRate: 8, respLatency: 12, throttledRate: 4 },
  infura:   { errorRate: 8, respLatency: 8,  throttledRate: 4 },
  drpc:     { errorRate: 8, respLatency: 4,  throttledRate: 2 },
};
(upstreams, ctx) =>
  upstreams.sortByScore((u) => weights[u.vendor] || BALANCED)
```

### Canary with cooldown

```js
(upstreams, ctx) =>
  upstreams
    .removeCordoned()
    .removeByErrorRate(0.5)
    .cooldown('1m')                                // keep excluded for 1m
    .sortByScore(BALANCED)
    .probeExcluded({ reAdmitAfter: '5m', maxConcurrent: 1 })
    .forceInclude('canary-rpc', 'tail')             // ALWAYS include canary
```

### Debug a flaky decision

```js
(upstreams, ctx) =>
  upstreams
    .removeByLag({ blockHead: 5 }).label('lag-filter')
    .sortByScore(BALANCED).label('score')
    .stickyPrimary().label('sticky')
    .dump('debug')                                  // logs full chain state
```

`label(...)` names a step so its rejections show `step: "lag-filter"` in
the decision record's `excluded[]` array.
