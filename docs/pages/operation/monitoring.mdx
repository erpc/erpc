# Monitoring and metrics

Network-level and upstream-level metrics are available via [Prometheus](https://prometheus.io/) and [Grafana](https://grafana.com/).

To enable metrics via config:
<Tabs items={["yaml", "typescript"]} defaultIndex={0} storageKey="GlobalConfigTypeTabIndex">
  <Tabs.Tab>
```yaml filename="erpc.yaml"
# ...
metrics:
  enabled: true
  listenV4: true
  hostV4: "0.0.0.0"
  listenV6: false
  hostV6: "[::]"
  port: 4001
  errorLabelMode: "verbose" # Optional: "verbose" (default) or "compact"
  histogramBuckets: "0.01,0.1,0.5,1,5,10,60,300" # Optional: custom histogram buckets
```
</Tabs.Tab>
  <Tabs.Tab>
```ts filename="erpc.ts"
import { createConfig } from "@erpc-cloud/config";
 
export default createConfig({
  // ...
  metrics: {
    enabled: true,
    listenV4: true,
    hostV4: "0.0.0.0",
    listenV6: false,
    hostV6: "[::]",
    port: 4001,
    errorLabelMode: "verbose", // Optional: "verbose" (default) or "compact"
    histogramBuckets: "0.01,0.1,0.5,1,5,10,60,300", // Optional: custom histogram buckets
  }
});
```
</Tabs.Tab>
</Tabs>

### Reducing Metrics Cardinality

eRPC provides two configuration options to help reduce metrics cardinality, which can significantly decrease the storage requirements and query performance of your monitoring system.

#### Error Label Mode

The `errorLabelMode` setting controls how detailed error information is included in metrics labels:

- `verbose`: Uses the full error message as labels (default for backward compatibility)
- `compact`: Uses only the error type as labels, reducing cardinality significantly
    
<Tabs items={["yaml", "typescript"]} defaultIndex={0} storageKey="GlobalConfigTypeTabIndex">
  <Tabs.Tab>
```yaml filename="erpc.yaml"
metrics:
  errorLabelMode: "compact" # "verbose" or "compact"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts filename="erpc.ts"
metrics: {
  errorLabelMode: "compact", // "verbose" or "compact"
}
```
  </Tabs.Tab>
</Tabs>

#### Histogram Buckets

You can customize histogram buckets to reduce cardinality and focus on relevant latency ranges:

<Tabs items={["yaml", "typescript"]} defaultIndex={0} storageKey="GlobalConfigTypeTabIndex">
  <Tabs.Tab>
```yaml filename="erpc.yaml"
metrics:
  histogramBuckets: "0.01,0.1,0.5,1,5,10,60,300"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts filename="erpc.ts"
metrics: {
  histogramBuckets: "0.01,0.1,0.5,1,5,10,60,300",
}
```
  </Tabs.Tab>
</Tabs>

Setting fewer buckets or focusing on relevant latency ranges can significantly reduce the number of time series stored in your monitoring system.

#### Experiment Dimensions

For canary/control experiment tracking, eRPC emits two low-cardinality labels on experiment metrics:

- `variant` from `ERPC_EXPERIMENT_VARIANT` (default: `default`)
- `release` from `ERPC_RELEASE_LABEL` (fallbacks: `ERPC_RELEASE`, `ERPC_IMAGE_TAG`, default: `unknown`)

Set these as deployment-level env vars (same values for all pods in a rollout) to avoid label explosion.

Refer to [erpc/docker-compose.yml](https://github.com/erpc/erpc/blob/main/docker-compose.yml#L4-L17) and [erpc/monitoring](https://github.com/erpc/erpc/tree/main/monitoring) for ready-made templates to bring up montoring.

### Available metrics

To get full list of available metrics check the source code of [erpc/telemetry/metrics.go](https://github.com/erpc/erpc/blob/main/telemetry/metrics.go).

![eRPC Grafana Dashboard](/assets/monitoring-example-erpc.png)

Here is a list of some of the most important metrics:

| Metric | Type | Description |
| --- | --- | --- |
| erpc_upstream_request_total | Counter | Total number of actual requests to upstreams. |
| erpc_upstream_request_duration_seconds | Histogram | Duration of upstream requests. |
| erpc_upstream_request_errors_total | Counter | Total number of upstream request errors. |
| erpc_upstream_request_skipped_total | Counter | Total number of upstream requests skipped by policy/failsafe. |
| erpc_network_request_received_total | Counter | Total number of requests received by a network. |
| erpc_network_successful_request_total | Counter | Total number of successful network requests. |
| erpc_network_failed_request_total | Counter | Total number of failed network requests. |
| erpc_network_multiplexed_request_total | Counter | Total number of requests served via multiplexing follower path. |
| erpc_network_request_duration_seconds | Histogram | End-to-end network request duration. |
| erpc_network_upstream_calls_per_request | Histogram | Upstream amplification per network request (how many upstream calls each request triggered). |
| erpc_network_attempt_reason_total | Counter | Non-baseline attempt causes (`retry`, `hedge`, `consensus`, `batch_fallback`, `fail_open`, `fire_and_forget`). |
| erpc_http_ingress_inflight | Gauge | Inflight HTTP ingress request count by route/method. |
| erpc_http_ingress_pre_forward_duration_seconds | Histogram | Time spent in ingress before forwarding/dispatch. |
| erpc_rate_limits_total | Counter | Unified rate-limit events (remote + local decisions). |
| erpc_rate_limiter_failopen_total | Counter | Fail-open events when limiter backend is unavailable/slow. |
| erpc_cache_set_success_total | Counter | Successful cache set operations. |
| erpc_cache_set_error_total | Counter | Failed cache set operations. |
| erpc_cache_set_skipped_total | Counter | Skipped cache set operations. |
| erpc_cache_get_success_hit_total | Counter | Cache hit operations. |
| erpc_cache_get_success_miss_total | Counter | Cache miss operations. |
| erpc_consensus_misbehavior_detected_total | Counter | Consensus misbehavior detections. |

#### PromQL examples

```bash
# Request rate per second by network over last 5 minutes
sum(rate(erpc_network_request_received_total{}[5m])) by (network)

# Total daily requests by project and network
sum(increase(erpc_network_request_received_total{}[24h])) by (project, network)

# Top 5 project and networks by request volume
topk(5, sum(rate(erpc_network_request_received_total{}[5m])) by (project, network))

# Error rate percentage by network and upstream
100 * sum(rate(erpc_upstream_request_errors_total{}[5m])) by (network, upstream) /
sum(rate(erpc_upstream_request_total{}[5m])) by (network, upstream)

# Top error types in the last hour
topk(10, sum(increase(erpc_upstream_request_errors_total{}[1h])) by (error))

# Missing data errors by network and upstream
sum(rate(erpc_upstream_request_missing_data_error_total{}[5m])) by (network, upstream)

# 95th percentile request duration by network
histogram_quantile(0.95, sum(rate(erpc_network_request_duration_seconds_bucket{}[5m])) by (le,network))

# Average request duration for eth_call methods
sum(rate(erpc_upstream_request_duration_seconds_sum{category="eth_call"}[5m])) by (network, upstream) /
sum(rate(erpc_upstream_request_duration_seconds_count{category="eth_call"}[5m])) by (network, upstream)

# Identify slow upstreams (avg duration > 500ms)
sum(rate(erpc_upstream_request_duration_seconds_sum{}[5m])) by (network, upstream) /
sum(rate(erpc_upstream_request_duration_seconds_count{}[5m])) by (network, upstream) > 0.5

# Cache hit ratio by network
sum(rate(erpc_cache_get_success_hit_total{}[5m])) by (network) /
(
  sum(rate(erpc_cache_get_success_hit_total{}[5m])) by (network) +
  sum(rate(erpc_cache_get_success_miss_total{}[5m])) by (network)
)

# Cache miss rate for eth_getBlockByNumber
rate(erpc_cache_get_success_miss_total{category="eth_getBlockByNumber"}[5m])

# Average upstream calls per request (amplification) by variant/release
sum(rate(erpc_network_upstream_calls_per_request_sum{}[5m])) by (project,network,variant,release) /
sum(rate(erpc_network_upstream_calls_per_request_count{}[5m])) by (project,network,variant,release)

# Attempt causes split by reason and experiment cohort
sum(rate(erpc_network_attempt_reason_total{}[5m])) by (project,network,reason,variant,release)

# Retry pressure by method for canary/control comparison
sum(rate(erpc_network_attempt_reason_total{reason="retry"}[5m])) by (category,variant,release)

# Local (non-remote) rate-limiting events by project/network
sum(rate(erpc_rate_limits_total{scope!="remote"}[5m])) by (project,network)

# Auth-origin rate-limiting events by auth strategy
sum(rate(erpc_rate_limits_total{origin="auth"}[5m])) by (project,auth)

# Remote rate limiting from upstreams
sum(rate(erpc_rate_limits_total{scope="remote"}[5m])) by (upstream)

# Block lag by network and upstream
max(erpc_upstream_block_head_lag) by (network,upstream)

# Finalization lag alert (lag > 5 blocks)
max(erpc_upstream_finalization_lag) by (network) > 5

# Block height difference between upstreams
max(erpc_upstream_latest_block_number) by (network) -
min(erpc_upstream_latest_block_number) by (network)

# Overall upstream health score
avg(erpc_upstream_score_overall) by (network, upstream)

# CORS issues by origin
sum(rate(erpc_cors_disallowed_origin_total{}[5m])) by (project, origin)

# Network block timestamp distance (how far behind is the latest block)
# All sources
erpc_network_latest_block_timestamp_distance_seconds{network=~"${network:regex}"}

# Only from EVM state poller (internal polling)
erpc_network_latest_block_timestamp_distance_seconds{network=~"${network:regex}",origin="evm_state_poller"}

# Only from network responses (what clients receive, including cached responses)
erpc_network_latest_block_timestamp_distance_seconds{network=~"${network:regex}",origin="network_response"}

# Alert if block timestamp is too far behind (> 30 seconds)
erpc_network_latest_block_timestamp_distance_seconds > 30
```
