## Footguns worth carrying into docs

1. `recheckInterval` is read with a `time.Duration` type assertion in every vendor (e.g. thirdparty/repository.go:56, thirdparty/quicknode.go:115); a raw YAML int/string that does not decode to `time.Duration` is silently ignored and the default is used.
2. `llama` and `routemesh` `OwnsUpstream` do not check their own scheme prefixes (llama.go:95-97, routemesh.go:135-140) — the shorthand works only because `convertUpstreamToProvider` runs first on any non-http scheme (common/defaults.go:1193-1206).
3. `dwellir` `OwnsUpstream` checks only `dwellir://`, not `evm+dwellir://` (dwellir.go:175); the `evm+` form still converts via defaults.go:1281.
4. `blockdaemon` requires `apiKey` even when `endpoint` is preset via overrides (blockdaemon.go:85-88).
5. `tenderly` chain-list URL is a hardcoded const — no `chainsUrl`-style override setting (tenderly.go:35).
6. `pimlico`/`routemesh` SupportsNetwork returns an error (not false) when `apiKey` is missing (pimlico.go:120-122, routemesh.go:52-55); `quicknode`/`chainstack` return false instead (quicknode.go:109-112, chainstack.go:83-86).
7. Provider-generated endpoints get `os.ExpandEnv` applied (thirdparty/provider.go:67-71) — literal `$VAR` in keys/URLs will be substituted.
8. `infura` Avalanche `/ext/bc/C/rpc` branch is unreachable (map values are `avalanche-mainnet`/`avalanche-fuji`, branch checks `ava-mainnet`/`ava-testnet`; infura.go:17-18,90-93).
